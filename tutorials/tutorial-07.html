🔧 Preparation

  1. Set up your project folder

    • If you haven’t already, you should have an R project already to work through your exercise. Below is an example of how to get started.

    • RStudio > New Project > New Directory > New Project

      Save it in a directory of your choosing.

      You can call the project what you like, e.g. etc5512.

    • Create folders called data and analysis (or similarly named) in the project root directory.

    • Create an exercise-07.Rmd under the analysis (or similarly named) folder.

  2. Getting the census data

    • Go to https://datapacks.censusdata.abs.gov.au/datapacks/ selecting the following options:

      • 2016 Census Datapacks
      • General Community Profile
      • All geographies
      • Vic

      The downloaded file would be called 2016_GCP_ALL_for_Vic_short-header.zip with size of 118 MB.

    • Unzip this file and save the folder in under data folder.

  3. Installing relevant R-packages

install.packages(c("remotes", "usethis", "here", "skimr", "unglue"))

🏋 Exercise 7B

Tidy your census data

Start with loading the tidyverse package and setting hte path ways to the data.

library(tidyverse)
library(unglue)
filter <- dplyr::filter
census_path <- here::here("data/2016_GCP_ALL_for_Vic_short-header/2016 Census GCP All Geographies for VIC/{geo}/VIC/")
STE_paths <- glue::glue(census_path, "2016Census_G17{alpha}_VIC_STE.csv", 
                        geo = "STE", alpha = c("A", "B", "C"))
SA1_paths <- glue::glue(census_path, "2016Census_G17{alpha}_VIC_SA1.csv", 
                        geo = "SA1", alpha = c("A", "B", "C"))

To load the data we will use the here function from the here package 📦. Note that we can use a function from a package without loading the package (i.e. without library(package)) by using the form package::function. The path argument parsed to here::here will be referenced with respect to Rproj is. This is particularly helpful if your project structure becomes deeply nested.

STE_G17Atidy <- pivot_longer(STE_G17A, -STE_CODE_2016,

🏃 Exercise 7C

Quality check your census data

  • According to the data from 7A, how many people are there in Victoria? If you check http://www.population.net.au, there are 6.03 million people in 2016 at Victoria. Does it look right? Are we missing some people in the data? If so, what kinds of people are we missing?
  • What is the minimum and maximum of values for count? Do the range and categories for each column look right?
  • Draw a barplot (using ggplot or otherwise) of the number people in Victoria by:
    1. sex
    2. age group
    3. income group
    4. sex and age group
    5. sex and income group
    6. age and income group
    7. sex, age and income group

Hint: click here for a code for the 7th barplot 

STE_G17tidy %>% 
   # remove `P` (should be a combination of female + male)
   filter(sex!="P") %>% 
   # make pretty age and income factors 
   mutate(income = fct_reorder(paste0(scales::dollar(income_min, big.mark = ""), "-", income_max), income_min),
          income = fct_recode(income, 
                              "Nil or negative" = "< $0",
                              "$3000 or more" = "> $3000"),
             age = fct_reorder(paste0(age_min, "-", age_max), age_min),
             age = fct_recode(age, "over 85" = ">85")) %>% 
   # plot
   ggplot(aes(x = age, y = count, fill = sex)) + 
      geom_col(position = "dodge") + 
   facet_wrap(~income) + 
   # let's not use the default theme
   theme_bw(base_size = 12) + 
   theme(axis.text.x = element_text(angle = 90, vjust = 0.3)) + 
   ggtitle("Income, sex & age distribution in Victoria from 2016 Census")
